# Pizza-Sales-SQL-Analysis
A comprehensive SQL project analyzing pizza sales data to uncover business insights and operational trends.

## 📌 Overview
This project demonstrates the use of SQL to analyze the sales data of a pizza restaurant. The primary objective is to extract actionable business insights regarding revenue, customer preferences, and operational trends by querying a relational database.

## Tools & Technologies
- **Language:** SQL
- **Concepts Applied:** Data Aggregation (`SUM`, `COUNT`, `AVG`), Joins, Grouping, Subqueries, Common Table Expressions (CTEs), and Window Functions (`RANK()`, `SUM() OVER()`).

## Files in this Repository
- `Pizza Sales Project SQL.sql`: The complete SQL script containing database creation, table setups, and all the analytical queries.
- `Questions.txt`: A structured list of the business questions answered in this project, categorized by difficulty (Basic, Intermediate, Advanced).
- `SQL Queries.pdf`: A cleanly formatted document showcasing the code structure and query progression.

## Key Insights Explored
The analysis is divided into three tiers of complexity:

**1. Basic Metrics:**
- Total orders placed and total revenue generated.
- Identification of the highest-priced pizza and the most common pizza size ordered.
- Top 5 most ordered pizza types by quantity.

**2. Intermediate Analysis:**
- Total quantity of each pizza category ordered.
- Distribution of orders by hour of the day to identify peak operational times.
- Category-wise distribution of pizzas.
- Average number of pizzas ordered per day.
- Top 3 most ordered pizza types based on overall revenue.

**3. Advanced Analytics:**
- Percentage contribution of each pizza type to total revenue.
- Cumulative revenue generated over time using window functions.
- Top 3 most ordered pizza types based on revenue within each specific pizza category.
